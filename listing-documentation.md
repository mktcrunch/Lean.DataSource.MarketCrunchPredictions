# MarketCrunch Predictions

## Introduction

The MarketCrunch Predictions dataset, by MarketCrunch AI, provides next-trading-day price
predictions for US equities and ETFs. Each record contains the predicted close price and
predicted percentage change for the following trading day, the model's confidence in that
prediction, and trailing directional-accuracy statistics (7-, 30-, and 90-day) for the
ticker as known at prediction time. Predictions are produced by an end-of-day model run and
are point-in-time: the accuracy statistics and create date reflect only information available
the trading day before the predicted date.

The initial release covers **2021-06-01 to 2026-06-01** across **246 symbols** — 193 US
large-caps (all current S&P 500 constituents) plus 53 ETFs spanning broad-index, sector,
leveraged/inverse, fixed-income and commodity funds — and is updated daily after market close.
Every series runs the full span; coverage is actively expanding.

## About the Provider

MarketCrunch AI, Inc. builds machine-learning models that forecast short-horizon equity price
movements. The predictions in this dataset are generated by the same production end-of-day
pipeline (model version `mc-eod-v1`) that powers the MarketCrunch product.

## Getting Started

The following snippets demonstrate how to subscribe to MarketCrunch Predictions data.

### Requesting Data

To add MarketCrunch Predictions data to your algorithm, call the **AddData** method. Save a
reference to the dataset **Symbol** so you can access the data later in your algorithm.

```python
class MarketCrunchPredictionsDataAlgorithm(QCAlgorithm):

    def initialize(self) -> None:
        self.set_start_date(2021, 6, 1)
        self.set_end_date(2024, 1, 1)
        self.set_cash(100000)
        self._symbol = self.add_equity("AAPL", Resolution.DAILY).symbol
        self._dataset_symbol = self.add_data(MarketCrunchPredictions, self._symbol).symbol
```
```csharp
public class MarketCrunchPredictionsDataAlgorithm : QCAlgorithm
{
    private Symbol _symbol, _datasetSymbol;

    public override void Initialize()
    {
        SetStartDate(2021, 6, 1);
        SetEndDate(2024, 1, 1);
        SetCash(100000);
        _symbol = AddEquity("AAPL", Resolution.Daily).Symbol;
        _datasetSymbol = AddData<MarketCrunchPredictions>(_symbol).Symbol;
    }
}
```

### Accessing Data

To get the current MarketCrunch Predictions data, index the current
[Slice](https://www.quantconnect.com/docs/v2/writing-algorithms/key-concepts/time-modeling/timeslices)
with the dataset **Symbol**. Slice objects deliver unique events to your algorithm as they
happen, but the Slice may not contain data for your dataset at every time step. To avoid
issues, check if the Slice contains the data you want before you index it.

```python
def on_data(self, data: Slice) -> None:
    if self._dataset_symbol in data:
        point = data[self._dataset_symbol]
        predicted_price  = point.prediction_price
        predicted_change = point.prediction_change
        confidence       = point.pred_confidence
        acc_30d          = point.last_30d_accuracy
```
```csharp
public override void OnData(Slice data)
{
    if (data.ContainsKey(_datasetSymbol))
    {
        var point = data[_datasetSymbol];
        var predictedPrice  = point.PredictionPrice;
        var predictedChange = point.PredictionChange;
        var confidence      = point.PredConfidence;
        var acc30d          = point.Last30DAccuracy;
    }
}
```

To iterate through all MarketCrunch Predictions objects in the current Slice, call the **Get**
method.

```python
def on_data(self, data: Slice) -> None:
    for dataset_symbol, point in data.get(MarketCrunchPredictions).items():
        predicted_change = point.prediction_change
```
```csharp
public override void OnData(Slice data)
{
    foreach (var kvp in data.Get<MarketCrunchPredictions>())
    {
        var datasetSymbol = kvp.Key;
        var point = kvp.Value;
        var predictedChange = point.PredictionChange;
    }
}
```

### Historical Data

To get historical MarketCrunch Predictions data, call the **History** method with the dataset
**Symbol**. If there is no data in the period you request, the history result is empty.

```python
# DataFrame
history_df = self.history(self._dataset_symbol, 100, Resolution.DAILY)

# Dataset objects
history_bars = self.history[MarketCrunchPredictions](self._dataset_symbol, 100, Resolution.DAILY)
```
```csharp
var history = History<MarketCrunchPredictions>(_datasetSymbol, 100, Resolution.Daily);
```

For more information about historical data, see
[History Requests](https://www.quantconnect.com/docs/v2/writing-algorithms/historical-data/history-requests).

### Remove Subscriptions

To remove a subscription to the MarketCrunch Predictions dataset, call the **RemoveSecurity**
method.

```python
self.remove_security(self._dataset_symbol)
```
```csharp
RemoveSecurity(_datasetSymbol);
```

Because MarketCrunch Predictions is a linked dataset, if you subscribe to predictions for
assets in a dynamic universe, remove the dataset subscription when the asset leaves your
universe. See
[Track Security Changes](https://www.quantconnect.com/docs/v2/writing-algorithms/algorithm-framework/alpha/key-concepts#05-Track-Security-Changes).

## Data Point Attributes

The MarketCrunchPredictions type exposes the following properties. The on-disk CSV column
order is: `create_date, ticker, prediction_date, prediction_price, prediction_change,
pred_confidence, last_7d_accuracy, last_30d_accuracy, last_90d_accuracy, model_version,
time, cutoff_date`.

| Column / Property | Python | Type | Description |
|---|---|---|---|
| `create_date` | `create_date` | date | Date the record was created/loaded (today's date for the initial bulk load). |
| `ticker` | `symbol` | Symbol | The security the prediction is linked to. |
| `prediction_date` | `prediction_date` | date | Trading day the prediction applies to. Exposed as the `PredictionDate` property. |
| `prediction_price` | `prediction_price` | decimal | Predicted close price for the prediction date. Also mapped to `Value`. |
| `prediction_change` | `prediction_change` | decimal | Predicted change vs. the prior close, as an absolute fraction (e.g. `0.0027` = +0.27%). |
| `pred_confidence` | `pred_confidence` | int | Model confidence score (0–100). 0 where the model did not emit a confidence. |
| `last_7d_accuracy` | `last_7d_accuracy` | decimal | Trailing 7-day directional (sign) accuracy %, as of the prior trading day's close. |
| `last_30d_accuracy` | `last_30d_accuracy` | decimal | Trailing 30-day directional accuracy %, as of the prior trading day's close. |
| `last_90d_accuracy` | `last_90d_accuracy` | decimal | Trailing 90-day directional accuracy %, as of the prior trading day's close. |
| `model_version` | `model_version` | string | Model version tag that produced the prediction (e.g. `mc-eod-v1`). |
| `time` | `time` | timestamp | Timestamp when the dataset/record was produced. |
| `cutoff_date` | `cutoff_date` | date | Last training date used for the next-day prediction (the prior trading day). LEAN **Time** (= **EndTime**, no period) is this date at 5:30pm ET — when the prediction becomes available. |

## Coverage & Point-in-Time Notes

- **Resolution:** Daily. **Timezone:** America/New_York (US equity market time). **Linked:** yes (mapped to the equity).
- **Point-in-time integrity:** `cutoff_date` and the trailing-accuracy columns reflect only
  information available as of the prior trading day's close (the day *before* `prediction_date`).
  Same-day actuals and accuracy are never used, so the dataset is free of look-ahead bias.
- **Sparsity:** All 246 series span the full 2021-06-01 → 2026-06-01 range; the data is sparse
  only in that non-trading days (weekends/holidays) have no rows — they simply do not appear in
  the file.
- **Confidence:** `pred_confidence` is 0 for a portion of historical rows where the model did
  not emit a score; this is expected and not a parsing error.
